纯文本提问处理流程：
```angular2html

文本理解：解析文本语义、识别问题意图和关键概念

文本检索：在文本向量索引中搜索相关内容

上下文构建：基于检索到的文本片段构建回答上下文

文本生成：生成基于文本上下文的回答

文本输出：返回纯文本格式的回答
```

文本+图像提问处理流程：
```angular2html
多模态理解：

文本解析：分析问题语义和意图

图像分析：识别视觉元素、物体、场景、文字等

意图融合：结合文本和视觉信息理解完整意图

跨模态检索：

文本检索：在文本索引中搜索相关内容

图像检索：在图像索引中搜索相似或相关图像

跨模态检索：使用融合向量搜索多模态内容

多模态上下文构建：整合文本和视觉信息构建丰富上下文

多模态生成：生成结合文本和视觉信息的回答

多模态输出：可能包含文本、图像引用或生成的视觉内容
```

##  技术处理差异

| 处理维度       | 纯文本提问     | 文本+图像提问                  |
| :------------- | :------------- | :----------------------------- |
| **输入编码**   | 单一文本编码器 | 文本编码器 + 视觉编码器        |
| **查询表示**   | 文本语义向量   | 融合的多模态向量               |
| **检索策略**   | 文本相似度搜索 | 跨模态语义匹配                 |
| **索引使用**   | 文本向量索引   | 文本、图像、多模态索引         |
| **上下文来源** | 文本片段       | 文本片段 + 图像特征 + 视觉概念 |
| **生成模型**   | 文本生成模型   | 多模态生成模型                 |
| **输出能力**   | 文本回答       | 文本回答 + 视觉引用/生成       |
